# dokploy-test

Local test environment for [Dokploy](https://dokploy.com) — an open source PaaS platform and alternative to Vercel, Netlify, and Heroku.

## Requirements

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) installed and running
- PowerShell (Windows) or Bash (Linux/macOS/WSL with Docker integration enabled)

## Quick Start

**1. Clone the repository**

```bash
git clone https://github.com/pcrisho/dokploy-test.git
cd dokploy-test
```

**2. (Optional) Configure ports and version**

```bash
cp .env.example .env
# Edit .env if you need to change ports or other options
```

**3. Start the environment**

On PowerShell (Windows):
```powershell
.\setup.ps1
```

On Bash (Linux/macOS/WSL):
```bash
bash setup.sh
```

**4. Open the dashboard**

```
http://localhost:3000
```

On first access, you'll be prompted to create an administrator account.

## Configuration

Copy `.env.example` to `.env` and adjust values as needed:

```env
# Dokploy version (latest, canary, v0.28.7, etc.)
DOKPLOY_VERSION=latest

# IP address for Docker Swarm. For local testing, 127.0.0.1 is fine.
ADVERTISE_ADDR=127.0.0.1

# Ports (change these if you have conflicts with other local services)
PORT_DASHBOARD=3000
PORT_HTTP=80
PORT_HTTPS=443
```

> `PORT_DASHBOARD` is completely free to use on any port. If you change `PORT_HTTP`/`PORT_HTTPS` to values other than 80/443, Traefik will continue to work but the URLs generated by Dokploy for your apps will assume standard ports — acceptable for local testing.

## Managing the Environment

### Pause (free up resources)

Stops all containers while preserving data:

```powershell
docker stack rm dokploy
```

### Resume

```powershell
.\setup.ps1
```

Since images are already cached and Swarm/network/secrets persist, the script detects existing state and deployment is nearly instant.

### Destroy everything

Removes the stack, secrets, network, and optionally data volumes:

```powershell
.\teardown.ps1
```

### Useful Commands

```powershell
# Check service status
docker stack services dokploy

# Real-time logs for Dokploy
docker service logs dokploy_dokploy -f

# Postgres logs
docker service logs dokploy_dokploy-postgres -f

# Traefik logs
docker service logs dokploy_dokploy-traefik -f
```

## Architecture

The stack deploys four services on a Docker Swarm overlay network:

| Service | Image | Host Port |
|---|---|---|
| `dokploy` | `dokploy/dokploy:latest` | `PORT_DASHBOARD` (3000) |
| `dokploy-postgres` | `postgres:16` | internal |
| `dokploy-redis` | `redis:7` | internal |
| `dokploy-traefik` | `traefik:v3.6.7` | `PORT_HTTP` (80), `PORT_HTTPS` (443) |

The Postgres password is randomly generated during setup and stored as an encrypted [Docker Secret](https://docs.docker.com/engine/swarm/secrets/) — it never appears in plain text in the compose file or `.env`.

## Why Swarm and Not Compose

Dokploy **requires** Docker Swarm to function. Internally, it uses `docker service create` to deploy your applications, creates overlay networks, and manages replicas. Without Swarm active, the dashboard starts but cannot deploy anything.

The `docker-compose.yml` file is deployed with `docker stack deploy` (Compose on Swarm), which allows maintaining the declarative nature of a compose file while Swarm runs underneath. **Do not use `docker compose up`** with this file — always use the setup scripts or `docker stack deploy` directly.
